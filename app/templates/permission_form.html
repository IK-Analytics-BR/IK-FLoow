Dicas para Nomes de Permissões

Use nomes descritivos e padronizados para facilitar o gerenciamento. Recomendamos seguir o formato módulo.ação, por exemplo:

  • financeiro.visualizar - Permite visualizar dados financeiros
  • estoque.editar - Permite editar dados de estoque
  • compras.aprovar - Permite aprovar pedidos de compra
Recomendado
  • Nomes concisos e significativos
  • Descrições claras e detalhadas
  • Permissões específicas para cada funcionalidade
  • Seguir um padrão de nomenclatura
Evitar
  • Nomes genéricos ou ambíguos
  • Descrições vagas ou incompletas
  • Permissões muito abrangentes
  • Duplicação de permissões existentes
Importante

Uma vez criada, a permissão poderá ser atribuída a usuários. Se precisar modificar o escopo de uma permissão, considere criar uma nova em vez de alterar uma existente, para evitar conceder acesso indevido a usuários que já possuem a permissão.
